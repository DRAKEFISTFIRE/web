04 — Craft
HOW I CODE

Code is a
craft,
not just a tool.

Every line I write follows principles that make software maintainable, scalable, and a pleasure to work with. These aren't rules I follow — they're habits I've built.

01
Clean Architecture
Strict separation of concerns — logic in composables, templates stay templates, styles stay styles. One layer, one job.
02
SOLID Principles
Single responsibility, open/closed, dependency inversion. Code that's easy to extend without touching what already works.
03
Security by Default
Sanctum/JWT auth, input sanitisation, CSRF protection, prepared statements. Security isn't day two — it's day one.
04
Performance Obsessed
Lazy loading, code splitting, RAF loops, debounced events. No layout thrash, no wasted re-renders, no bytes lost on the wire.
05
DRY & Composable
Shared composables and utilities over copy-paste. Write once, use everywhere — with zero hidden coupling between modules.
06
Git Discipline
Conventional commits, feature branches, atomic PRs. Every change is intentional, traceable, and rollback-safe.
07
Accessible by Default
Semantic HTML, ARIA roles, keyboard nav, WCAG AA contrast. The web should work for everyone — full stop.
